## 1. Executive verdict

**Conditional-Go.**

초안은 구조적으로 공모전 요구에 부합하고, 포지셔닝(공공 법령데이터 기반 안내 도구)도 GEMINI 액션플랜과 일치합니다. 그러나 현재 구현 상태와 초안 본문 사이에 *현재형/계획형 시제 혼동*이 다수 존재하며, 일부 법령 조문 인용은 검증되지 않았습니다. 이 두 가지 문제를 해결하지 않고 제출하면, 데모를 직접 검증하는 심사위원 또는 시연 단계에서 신뢰도 손실이 발생합니다.

핵심 문제 세 가지:
1. **시제 혼동**: 인용 배지·5행위 라벨·EN 토글은 13절에서 "1~2주 차 작업"으로 명시되었으나, 6절·8절은 이미 구현된 것처럼 기술됨.
2. **법령 조문 번호 미검증**: 부록 A 대조표는 조문 호 번호를 단정적으로 제시. 부록 말미에 "재확인 필요"가 있으나, 본문 5절·11절은 이미 검증된 것처럼 인용.
3. **39 vs 56 데이터 정합성**: `visa_data.json`은 실제로 56건(시나리오·FAQ 포함). 초안은 일관되게 "39종"만 제시하며, 이 차이를 설명하지 않음. 심사위원이 데이터를 직접 열어보면 즉시 발견됨.

위 세 항목을 9절(편집 권고)에 따라 정리하면 Go로 전환 가능합니다. 정리 없이 제출하면 No-Go 위험이 큽니다.

## 2. Fit with the 법령데이터 활용 아이디어 공모전

### 2.1 적합 요소
- 법제처 `law.go.kr`·`open.law.go.kr` 데이터를 명시적 출처로 호명 — 공모전 핵심 키워드와 정합.
- "법령 → 매뉴얼 → 사용자 시나리오" 변환 흐름(5.2절)은 공모전이 요구하는 *공공 법령 데이터의 시민 활용* 서사를 구체화.
- 출입국관리법 시행령 별표라는 *명확히 식별 가능한 법령 데이터*를 대상으로 함. 공모전 평가에서 대상 법령 데이터의 모호함은 흔한 감점 요인인데, 이 부분은 통과.
- 39종 체류자격이라는 도메인은 공모전 응모자 풀에서 차별성이 명확.

### 2.2 부적합·약한 요소
- **`open.law.go.kr` API 실연동 부재**. 현재 `index.html`·`ai.html` 어디에도 `law.go.kr` 링크/API 호출 흔적이 없음(grep 검증). 초안 5.1절은 API 활용을 현재형으로 기술하나, 실제로는 로드맵(13절·12.2절). 심사위원이 "지금 API를 호출합니까?"를 물으면 답이 궁색해짐.
- **법령 조문 직접 인용의 가시화 부재**. 초안 6.1절은 "각 카드에는 법령 근거 조문과 매뉴얼 인용 페이지가 표기됩니다"라고 단정하나, 13절은 "절차 카드별 법령 조문·매뉴얼 인용 배지 추가"를 1주 차 작업으로 명시. 동일 문서 내 모순.
- **공모전이 요구하는 *법령데이터의 차별적 활용*에 대한 자기 변별점이 약함**. "다국어 접근성"은 공모전 평가에서 부수적 요소이며, 핵심 경쟁점이 되기 어려움. 법령데이터 활용 자체의 깊이(예: 시행령 개정 이력 추적, 별표 자동 파싱, 조문 단위 cross-reference)에 더 무게를 실어야 경쟁력이 생김.

**판정**: 적합도 자체는 높지만, *현재 구현 상태와 초안 주장 사이의 간극*이 공모전 적합성을 잠식함.

## 3. Legal overclaim risk review

### 3.1 비교적 안전한 부분
- 9.1절 고지 정형구는 GEMINI 액션플랜의 권고 표현과 일치(법률 자문/행정 대행/신청 접수 비해당, 1345·관할청 확인).
- 9.2절은 "단정 표현 금지" 원칙을 명시.
- 11.2절 D-10 데모 응답은 "단, 다음 사항은 매뉴얼이 일률적으로 규정하지 않으므로 창구 확인이 필요합니다"로 적절히 종결.
- 10절은 개인정보 비수집·민원대행 미제공·유료 추천 비연계를 명시 — 공공섹터 신뢰성 강화.

### 3.2 잔존 위험
- **8.3절 "단정 표현 자동 차단"**: "받을 수 있습니다", "가능합니다", "됩니다" 등을 시스템 수준에서 차단한다고 단정. 코드 검증 결과 자동 차단 로직은 확인되지 않음. 이는 *기능 자체에 대한 단정*으로, 심사위원이 데모를 시도해 "가능합니다"가 한 번이라도 나오면 신뢰가 무너짐. 계획·지향으로 표현해야 함.
- **8.3절 "RAG 기반 검색"**: 백엔드(`moonshot_backend_fastapi.py:43-332`)에 Supabase pgvector RAG가 실재함은 확인. 단, 사용자에게 *현재 동작하는* 검색이 매뉴얼의 어느 청크에서 나온 답인지를 화면상으로 노출하는지는 별개 문제. UI 인용 표시가 없다면 "근거 인용"은 백엔드 내부 동작으로만 존재하고, 사용자 신뢰 구조로는 미완.
- **6.1절 "각 카드에는 법령 근거 조문과 매뉴얼 인용 페이지가 표기됩니다"**: 현재 `index.html`에 해당 표기 없음. 사실과 다름.
- **11.2절 D-10 데모 [2단계] 서류 목록 — "졸업증명서, 재정 능력 입증 서류"**: 일반 안내로 적정. 단, "신청서(통합신청서)"·"수수료" 등 항목은 매뉴얼의 정확한 페이지 인용 없이 일반화됨. 대조표(부록 A)의 D-10 행과 매핑이 필요.
- **부록 A "출입국관리법 시행령 별표1의2 제28호 = D-10"**: 호 번호가 실제 시행령 최신 본과 일치하는지 미검증. 부록 말미에 "재확인 필요"가 있으나, 본문 11.2절 [3단계] AI 응답에는 해당 호 번호가 단정적으로 인용됨. 본문에 들어간 호 번호는 검증 전 *임시값*이라는 표시가 없음.
- **14절 "오픈소스 기반"**: 저장소 라이선스 파일 미확인 시 이는 단정 위험. 라이선스 파일 존재·종류 확인 후에만 사용해야 함.

## 4. Public-data and legal-data utilization review

### 4.1 활용 깊이 평가
- **현재 시점의 실제 활용**: 매뉴얼 텍스트(`docs/sajeung-manual.md`, `docs/ceryu-manual.md`)를 RAG 컨텍스트로 사용. `visa_data.json` 39종 카탈로그(단, 실제 56건). `law.go.kr` API 미연동.
- **초안이 주장하는 활용**: `open.law.go.kr` 조문 단위 매핑, 39 카탈로그 자동 동기화, 법령 개정 자동 감지.
- **간극**: 초안 4.1절·5.1절은 활용을 *현재 진행형*에 가까운 시제로 기술. 실제로는 로드맵(12.2절·13절). 심사위원이 코드를 열어보면 *법령 데이터 활용은 매뉴얼 PDF 인용 + 카탈로그 데이터 + 백엔드 RAG*에 한정됨을 확인하게 됨.

### 4.2 보강 제안
- 5절을 *현재 활용 / 단기 활용 / 중기 활용*의 3단계 표로 분리할 것. 현재 구현되지 않은 항목은 표 안에서 명시적으로 "계획"으로 표기.
- "법령데이터 활용"의 차별점을 *조문 단위 cross-reference*나 *시행령 별표의 시민 인터페이스화*로 구체화. 단순 "인용 표기"는 변별력 약함.
- `law.go.kr` 링크를 데모 화면에 *직접 클릭 가능한 형태*로 노출하는 미니 데모(데모 영상 한 장면)를 만들 것. 이것 하나만으로 "법령데이터 활용" 주장이 시각적으로 보증됨.

### 4.3 라이선스·인용 정책
- 초안에 공공누리(KOGL) 라이선스 명시 부재. `law.go.kr` 데이터의 공공누리 적용 여부와 매뉴얼 PDF 재배포 정책을 한 줄 추가 필요. 공모전 심사에서 라이선스 처리 미명시는 감점 요인.

## 5. D-2 → D-10 demo scenario review

### 5.1 시나리오 구성의 신빙성
- 페르소나 설정(중국 국적 D-2 컴퓨터공학 전공 졸업, 5월 만료) — 현실성 충분.
- 흐름(카탈로그 탐색 → 절차 안내 → AI 입력 → 관할관서 조회) — 사용자 동선이 자연스러움.
- 시연 시간 추산: 약 60~90초 — 공모전 1분 데모 영상 권장 길이에 부합.

### 5.2 안전성
- AI 응답이 자격 충족을 단정하지 않음. "전공·학위 요건의 구체적 충족 여부, 재정 요건의 세부 금액, 졸업 후 경과 기간"을 *창구 확인 사항*으로 분리한 처리는 적절.
- "1345 또는 관할 출입국·외국인청"으로 종결 — 안전 기준 충족.

### 5.3 사실 검증 필요 항목
- "D-10(구직)은 일반적으로 한국 내 4년제 대학 졸업자 또는 이에 준하는 학위 소지자를 대상" — 실제 D-10은 *점수제*가 적용되며, *국내 학위*뿐 아니라 *해외 학위 + 추가 요건* 경로도 존재. 초안의 "국내 4년제" 표현은 너무 좁음. 매뉴얼 원문 재확인 후 "국내 학위 또는 일정 요건을 충족하는 해외 학위 소지자"로 완화 필요.
- "체류기간 만료 전에 관할 출입국·외국인청에서 진행" — 정확하나, *만료 4개월 전 신청 가능*과 같은 시기 제약은 명시되지 않음. 제출 데모에서는 시기 제약을 *창구 확인*으로 떼어두는 것이 안전.
- 서류 목록("신청서, 여권, 외국인등록증, 졸업증명서, 재정 능력 입증, 수수료") — 일반 안내로 적정. 단, *어학 능력 증명(TOPIK 등) 또는 점수표* 가능성을 누락한 점은 주의. "추가 서류 가능성 있음"을 명시한 점은 좋음.

### 5.4 데모 화면 캡처 시 주의
- AI 응답 박스에 *반드시* 출처 인용 + 1345 안내가 노출되는 화면을 캡처할 것. 13절 1주 차 작업("AI 응답 말미 정형구 자동 부착")이 데모 영상 촬영 전에 완료되어야 함.
- 데모 시점에 인용 배지가 없으면, 심사위원이 데모와 초안 본문 사이의 불일치를 발견함.

## 9. Top 5 edits to improve award potential

> 우선순위 순. 위에서부터 고치면 시간 대비 효과가 가장 큼.

### E1. 시제 정렬 — 현재 vs 계획의 분리
- 6절·8.1·8.3절을 *현재 동작*과 *1~2주 차 보강*으로 분리하여 표기.

- **효과**: 심사위원이 가장 먼저 확인할 자료의 정확성 확보.

### E3. `law.go.kr` 데모 한 장면 추가
- 데모 영상 또는 발표 자료에 *시행령 별표1의2 페이지를 직접 클릭해 여는 한 장면*을 포함.

- **효과**: 심사위원이 데이터를 열어볼 때 발견하는 불일치를 *사전 설명*. 약점을 투명성으로 전환.
