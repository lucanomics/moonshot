## 3. 공통 취향 요약
레퍼런스 전반에서 관찰되는 취향은 다음과 같다.

- **타이포그래피 주도형 구성**: 시각적 중심을 이미지가 아니라 텍스트 위계와 활자 밀도에 둔다.
- **에디토리얼 그리드 기반**: 카드 나열형 UI보다 열/행, 인덱스, 리스트 구조를 선호한다.
- **절제된 개성**: 장식이 많지 않지만 간격, 라인, 비례, 타이포 디테일로 캐릭터를 만든다.
- **따뜻하지만 귀엽지 않은 표면감**: 차갑게 기술적이거나 과도하게 감성적인 톤 대신, 온도를 약간 올린 중성 계열을 선호한다.
- **스튜디오식 긴장감**: 명확한 정보 구조 위에 정교한 편집 감각을 얹는 방식.
- **웹사이트 우선 사고**: 모바일 앱 패턴(큰 카드, 둥근 컴포넌트, 챗 UI)보다 데스크톱 웹 편집 체계 중심.

## 4. 레이아웃 취향 분석

### 4.1 카드 최소화와 인덱스화
사용자 취향은 “많은 카드”보다 “적은 블록 + 정교한 행 구조”에 가깝다. 즉, 정보 단위를 박스에 과도하게 격리하지 않고, 얇은 구분선과 여백으로 구획한다.

### 4.2 에디토리얼 그리드
단일 컬럼 스크롤보다 다음 구조를 선호한다.
- 좌측: 섹션 인덱스/내비게이션
- 중앙: 본문 정보(타이틀, 설명, 상태)
- 우측: 메타 정보(업데이트, 카테고리, 참조)

이는 문서적 신뢰감과 탐색 효율을 동시에 제공한다.

### 4.3 리듬 있는 밀도
콘텐츠 밀도가 낮아 텅 빈 랜딩페이지도, 반대로 높은 밀도의 법률 DB 느낌도 피해야 한다. 핵심은 “읽기 가능한 고밀도”이며, 행 간격과 타이포 대비로 리듬을 만든다.

## 5. 타이포그래피 취향 분석

### 5.1 강한 위계
헤드라인/섹션 타이틀/행 텍스트/보조 메타의 단계가 명확해야 한다. 크기 차이뿐 아니라 굵기, 자간, 대소문자 규칙, 숫자 정렬(탭형 숫자) 등으로 계층을 강화한다.

### 5.2 정보 신뢰를 만드는 활자 태도
법률·행정 성격의 데이터에서는 “장식적 개성”보다 “정확해 보이는 문장 운용”이 중요하다. 다만 관료적 포털처럼 무표정하지 않게, 제목부에서 편집적 캐릭터를 부여한다.

### 5.3 한국어/영문 혼용 최적화
Paradiso 맥락상 한국어 중심이지만, 비자 코드/영문 고유명사/기관명이 함께 등장한다. 따라서 한글 본문 가독성과 라틴 문자 숫자 정렬의 균형이 핵심이다.

## 6. 색상 및 표면감 분석

### 6.1 웜 뉴트럴 기반
기본 표면은 백색 일변도보다 약간 따뜻한 오프화이트·웜그레이 계열이 적합하다. 이는 차가운 데이터 인상을 완화하되, 귀엽거나 라이프스타일 브랜드처럼 보이지 않게 한다.

### 6.2 저채도 대비 + 기능적 포인트
포인트 컬러는 다채색 확장보다 소수의 기능적 강조색(링크, 상태, 경고)에 집중한다. 메인 정체성은 채도가 아니라 타이포와 그리드에서 확보한다.

### 6.3 얇은 경계선의 적극 활용
카드 그림자 대신 얇은 디바이더 라인으로 위계와 구역을 정의한다. 이는 “index-like rows” 감성을 강화한다.

## 7. 인터랙션/모션 취향 분석

### 7.1 절제된 반응성
호버, 포커스, 확장 동작은 존재하되 과시적 모션은 지양한다. 빠른 응답, 짧은 거리, 명확한 상태 변화가 중요하다.

### 7.2 텍스트 중심 인터랙션
버튼 애니메이션보다 행 하이라이트, 링크 언더라인 전환, 정렬/필터 상태 표기처럼 **읽기 흐름을 돕는 마이크로 인터랙션**이 우선이다.

### 7.3 탐색 우선 모션
모션은 브랜딩 쇼케이스가 아니라 탐색 비용 감소에 기여해야 한다. 예: 섹션 점프 시 위치 인지, 필터 적용 시 결과 영역의 안정적 갱신.

## 8. Paradiso에 적용할 원칙

1. **타이포가 레이아웃을 이끈다**: 컴포넌트 모양보다 텍스트 위계 설계가 선행되어야 한다.
2. **카드 대신 인덱스 행 구조를 기본값으로 둔다**.
3. **얇은 디바이더와 정교한 여백으로 구획한다**.
4. **웹사이트 우선 편집 체계**(특히 데스크톱)로 화면을 설계한다.
5. **웜 뉴트럴 바탕 + 저채도 포인트**로 표면 온도를 조정한다.
6. **모션은 기능 중심으로 절제**한다.
7. **시빅-테크 신뢰감 + 스튜디오 편집감**을 동시에 유지한다.

## 9. Paradiso에서 피해야 할 방향
다음 방향은 본 취향과 충돌하므로 명시적으로 배제한다.

- 박물관 아카이브 같은 정적 전시 톤
- 범용 AI 챗봇 UI(대화 버블 중심)
- 제너릭 SaaS 랜딩 페이지 미학
- 정부 포털 클론 스타일
- 모바일 검색 앱 중심 구조
- K-pop/팬덤 감성의 과도한 장식성
- 법률 데이터베이스형 과밀 테이블 UX
- 카드가 과도하게 많은 대시보드형 화면

## 10. 새 디자인 컨셉: Editorial Civic Index

### 10.1 컨셉 정의
**Editorial Civic Index**는 “행정 정보를 읽기 좋게 편집한 공공성 있는 웹 인덱스”를 의미한다. 핵심은 ‘검색 도구’가 아니라 ‘신뢰 가능한 읽기/탐색 환경’이다.

### 10.2 시각 원칙
- 강한 제목부와 명확한 서브헤드
- 목록 중심 본문(행/열/인덱스)
- 얇은 선 기반 구분
- 절제된 강조색
- 문서형 리듬을 가진 섹션 전개

### 10.3 경험 원칙
- 사용자는 빠르게 스캔하고, 필요한 맥락을 즉시 확인할 수 있어야 한다.
- 정보 단위는 “읽히는 순서”에 맞춰 편집되어야 한다.
- UI의 존재감보다 정보 이해 속도를 우선한다.

## 11. 기존 Soft Source Archive와의 차이

- **Soft Source Archive**: 부드럽고 아카이브적인 보관/감상 톤, 상대적으로 완만한 위계.
- **Editorial Civic Index**: 강한 정보 위계, 인덱스 중심 탐색, 시빅-테크 신뢰를 전면화.

전환의 핵심은 “분위기”가 아니라 “정보 구조와 읽기 프로토콜”의 변경이다.

## 12. Figma Make용 최종 지시문
다음 문장을 Figma Make 프롬프트의 기본 지시문으로 사용한다.

> Paradiso를 한국 비자·체류 정보를 위한 typography-led editorial civic-tech web index로 리디자인하라. 카드 중심 UI를 줄이고, 얇은 디바이더와 index-like row 시스템, 명확한 타이포 위계를 사용하라. 레이아웃은 website-first 편집 구조(좌측 인덱스/중앙 본문/우측 메타)를 기본으로 하며, 색상은 warm neutral 기반에 저채도 기능 포인트만 허용한다. 결과물은 museum archive, generic AI chatbot, generic SaaS landing, government portal clone, mobile app search UI, K-pop/fan style, legal DB 과밀 테이블처럼 보이면 안 된다.

## 13. 구현 전 체크리스트

- [ ] 메인 화면이 카드 그리드보다 인덱스 행 구조에 가깝다.
- [ ] 헤드라인/섹션/본문/메타의 타이포 위계가 즉시 구분된다.
- [ ] 디바이더 두께와 여백 리듬이 일관적이다.
- [ ] 웜 뉴트럴 표면이 적용되었고 과한 채도 사용이 없다.
- [ ] 모션은 짧고 기능 중심이며 시선 분산을 유발하지 않는다.
- [ ] 데스크톱 웹 기준의 편집 레이아웃 완성도가 확보되었다.
- [ ] 정부 포털/챗봇/SaaS 랜딩/모바일 앱 톤으로 오인되지 않는다.
- [ ] 정보 탐색 속도(스캔→선택→확인)가 기존안보다 개선되었다.
