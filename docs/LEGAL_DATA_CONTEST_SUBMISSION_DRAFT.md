# Paradiso — 제2회 법령데이터 활용 아이디어 공모전 출품 초안

> 본 문서는 법제처 주관 제2회 법령데이터 활용 아이디어 공모전 제출을 위한 한국어 출품 초안입니다.
> 코드·데이터·UI 변경을 일체 수반하지 않으며, 출품 내러티브·기능 설명·데모 시나리오를 정리합니다.
> 최종 제출 전 담당자가 직접 검토·수정하여 사용하십시오.

---

## 1. 한 줄 소개

**Paradiso**는 출입국관리법령과 출입국·외국인정책본부 실무 매뉴얼을 근거로, 외국인이 한국 비자·체류 절차를 *출처를 확인하며* 이해할 수 있도록 돕는 공공 법령데이터 기반 안내 도구입니다.

---

## 2. 문제 정의

한국 체류 외국인은 250만 명을 넘었지만, 비자·체류 정보는 다음 네 곳에 분산되어 있습니다.

1. **출입국관리법·시행령·시행규칙** — 국가법령정보(`law.go.kr`, `open.law.go.kr`)에 공개되어 있으나 행정 한국어로 작성됨
2. **사증민원 매뉴얼·체류민원 매뉴얼** — PDF 형태로 공표되나 외국인이 직접 탐색하기 어려움
3. **하이코리아(hikorea.go.kr)** — 공식 안내 포털이나 언어·시나리오 깊이가 제한적
4. **관할 출입국·외국인청 창구** — 정확하지만 접근성이 낮고, 대기 부담이 큼

외국인은 이 네 출처를 스스로 교차 확인해야 합니다. 한국어와 행정 지식이 부족한 상황에서 이 과정은 잘못된 신청, 미신고 과태료, 체류자격 상실 같은 손실로 이어집니다.

**핵심 문제**: 공공 법령데이터는 이미 개방되어 있습니다. 그러나 외국인이 자기 상황에 맞는 조문과 절차를 *직접 추적할 수 있는 단일 인터페이스가 없습니다*.

## 3. 대상 사용자

| 페르소나 | 대표 상황 | 주요 체류자격 |
|---|---|---|
| 외국인 유학생 | 졸업 후 구직 기간 체류 희망 | D-2 → D-10 |
| 외국인 근로자 | E-9 만료 전 갱신·변경 탐색 | E-9, E-7 |

| 재외동포 | 재외동포 자격 요건·갱신 | F-4 |

본 서비스는 이들이 *어떤 절차가 필요한지 이해하는 단계*를 보조합니다. 자격 판단·신청 대행·법률 자문은 본 서비스의 범위가 아닙니다.

## 4. 활용 공공데이터

### 4.1 법제처 법령데이터 (공모전 핵심)

| 데이터 | 출처 | 활용 방식 |
|---|---|---|
| 출입국관리법 | `law.go.kr` | 체류자격 정의·수권 근거 조문 인용 |
| 출입국관리법 시행령 별표1·별표1의2 | `open.law.go.kr` API (연동 예정) | 39종 법정 체류자격 카탈로그 구조화 기반. 조문 단위 API 직접 연동은 출품 로드맵 항목 |
| 출입국관리법 시행규칙 | `law.go.kr` | 서류·수수료 요건 근거 인용 |
| 출입국관리법 관련 고시·예규 | `law.go.kr` | 특정 자격 세부 기준 인용 |

법제처 `open.law.go.kr` API는 조문 단위 기계판독 가능 데이터를 제공합니다. 현재 Paradiso는 법령 조문을 출처 인용 형태로 안내에 포함하며, `open.law.go.kr` API를 통한 조문 자동 매핑은 출품 로드맵(중기 계획)에 포함됩니다.

### 4.2 출입국·외국인정책본부 실무 매뉴얼

| 자료 | 버전 | 역할 |
|---|---|---|
| 사증민원 매뉴얼 | 2026.04.14 | 사증 발급·발급인정서 절차 기준 |
| 체류민원 매뉴얼 | 2026.04.14 | 외국인등록·체류연장·자격변경·신고 기준 |

매뉴얼은 법령의 *운영 해석본*입니다. 법령 조문이 규정 골격을 제공하면, 매뉴얼은 창구 심사 실무 기준을 제공합니다. Paradiso는 두 출처를 연계 인용합니다.

### 4.3 보조 공공데이터

- 하이코리아(hikorea.go.kr) — 관할관서 정보, 공식 서식 링크
- 1345 외국인종합안내센터 — 최종 확인 채널 연결
- 법무부 출입국·외국인정책 통계연보 — 체류 외국인 현황(250만 명 이상) 통계 인용

## 5. 법령데이터 활용 방식

Paradiso의 법령데이터 활용은 세 단계로 구분됩니다.

### 5.1 구조화 단계
출입국관리법 시행령 별표1·별표1의2의 39종 법정 체류자격 분류를 카탈로그 데이터로 구조화합니다. 현재 각 항목에는 매뉴얼 인용 출처와 데이터 캡처 일자가 포함되며, 조문 번호 단위 매핑은 출품 준비 작업으로 보강 예정입니다.

### 5.2 연계 단계
사용자가 특정 체류자격을 조회하면 다음 세 층위를 동시 제시합니다.

```
[법령 조문]  →  [매뉴얼 절차 안내]  →  [관할관서·서식 링크]
출입국관리법      사증민원·체류민원       히코리아·1345
시행령 별표1       매뉴얼 인용 페이지        확인 채널
```

### 5.3 AI 검색 단계
사용자가 자연어로 상황을 입력하면 RAG(검색 증강 생성) 구조가 사증민원·체류민원 매뉴얼 텍스트에서 관련 구절을 검색한 뒤, 근거를 인용한 답변을 생성합니다. AI는 매뉴얼에 명시되지 않은 내용은 추정하지 않고 "확인 필요"를 안내합니다.

## 6. 서비스 핵심 기능

### 6.1 39종 체류자격 카탈로그
출입국관리법 시행령 별표에 따른 39종 법정 체류자격(단기·장기·영주·특수 포함)을 카테고리·키워드로 탐색합니다. 구조화된 데이터에는 법정 39종 외에 시나리오·FAQ 등 보조 안내 항목이 포함되며, 법정 체류자격과 구분하여 제공됩니다. 각 카드에 법령 근거 조문과 매뉴얼 인용 페이지 표기는 출품 준비 작업(1주 차)으로 적용 예정입니다.

### 6.2 절차 안내 (5행위 구분)
사증민원 매뉴얼과 체류민원 매뉴얼이 구분하는 다섯 가지 행정행위를 명확히 분리하여 안내합니다.

| 행위 | 담당 기관 | 적용 국면 |
|---|---|---|
| 사증 발급 | 재외공관 | 입국 전 |
| 외국인등록 | 출입국·외국인청 | 입국 후 90일 이내 |
| 체류기간 연장 | 출입국·외국인청 | 기간 만료 전 |
| 체류자격 변경 | 출입국·외국인청 | 자격 요건 변경 시 |
| 각종 신고 | 출입국·외국인청 | 체류지 변경·근무처 변경 등 |

### 6.3 관할관서 조회
시·도/시·군·구 기준으로 관할 출입국·외국인청·사무소·출장소를 조회합니다. 외국인이 *어디로 가야 하는지*까지 안내합니다.

### 6.4 Paradiso AI (자연어 상황 입력)
사용자가 자기 상황을 자연어로 입력하면, RAG 기반으로 매뉴얼 텍스트에서 관련 구절을 검색해 출처 인용과 함께 안내합니다. 모든 응답 말미에 1345·하이코리아 확인 안내를 자동 부착하는 기능은 출품 준비 작업(1주 차)으로 적용 예정입니다.

### 6.5 K-ETA·사증면제 안내
입국 전 단계 — 사증면제협정 해당 여부, K-ETA 필요 여부를 국적·방문 목적 기준으로 안내합니다.

## 7. AI/RAG 구조 요약

```
사용자 질의
    │
    ▼
[질의 분류]
 ─ 체류자격 조회 / 절차 안내 / 관할관서 / 일반 질의
    │
    ▼
[검색 (Retrieval)]
 ─ 벡터 검색: 사증민원 매뉴얼 + 체류민원 매뉴얼 텍스트 청크
 ─ 구조 검색: visa_data.json (39종 카탈로그)
 ─ 법령 인용: open.law.go.kr 조문 연동 (출품 로드맵 예정)
    │
    ▼
[생성 (Generation)]
 ─ 인용된 근거 구절을 컨텍스트로 LLM 답변 생성
 ─ 매뉴얼에 없는 내용: 추정 금지, "확인 필요" 처리
 ─ 불확실성 표기: 케이스가 명시되지 않은 경우 명시
    │
    ▼
[출력]
 ─ 답변 본문
 ─ 출처 인용 (매뉴얼 페이지/조문 번호/캡처 일자)
 ─ 1345·하이코리아 확인 안내 (자동 부착)
```

AI는 합격·승인·기간을 단정하지 않도록 설계되었으며, 응답 품질을 정기 점검합니다. 개별 심사관의 재량 판단이 항상 존재하며, 이를 숨기지 않습니다.

## 8. 근거 중심 답변 설계

Paradiso의 답변은 다음 세 가지 계층으로 구성됩니다.

### 8.1 출처 인용 의무
모든 절차 안내는 다음 세 항목을 동시 제시합니다.
- **(a) 인용 근거**: 매뉴얼 페이지 또는 법령 조문 번호
- **(b) 데이터 캡처 일자**: 2026.04.14 사증·체류 매뉴얼 기준
- **(c) 확인 채널**: 1345 또는 hikorea.go.kr 링크

예시:
> "D-10(구직) 체류자격으로의 변경은 일반적으로 국내 학위 취득자 또는 일정 요건을 충족하는 해외 학위 소지자를 대상으로 합니다.
> **출처**: 체류민원 매뉴얼 2026.04.14, D-10 항목 (관련 시행령 별표 조문 번호는 법령 원문 검증 후 확정 예정).
> 개별 심사 기준은 관할 출입국·외국인청에 따라 다를 수 있습니다. 최종 확인은 반드시 1345 또는 관할청에 하시기 바랍니다."

### 8.2 불확실성 표기 원칙
매뉴얼이 케이스를 명시하지 않거나 사용자 상황이 경계에 있을 경우, AI는 추정을 금지하고 다음 정형구를 사용합니다.

> "이 사안은 매뉴얼에서 일률적으로 규정되어 있지 않습니다. 관할 출입국·외국인청 또는 1345(외국인종합안내센터, 다국어 지원)에 확인하십시오."

### 8.3 AI 환각(Hallucination) 통제
- RAG 기반 검색 설계: 매뉴얼 텍스트에서 검색한 구절을 컨텍스트로 활용하여 외부 학습 지식만으로 생성되는 응답을 억제하는 구조
- 인용 근거 중심 답변 원칙: 검색 결과가 없을 경우 "관련 근거를 찾지 못했습니다"로 종결하는 것을 설계 목표로 하며, 응답 품질을 정기 점검합니다
- 단정 표현 회피 프롬프트 설계: "받을 수 있습니다", "가능합니다", "됩니다" 등 승인 암시 표현을 회피하도록 프롬프트를 구성하며, 출품 데모 기준으로 응답을 우선 점검합니다

## 9. 법적 리스크 통제 방안

### 9.1 법률 자문 비해당 명시
본 서비스는 출입국관리법 및 매뉴얼의 일반 정보를 안내하는 행정 가이드 보조 도구입니다. 법률 자문(legal advice), 법률 대리(legal representation), 행정 처분(administrative decision)을 제공하지 않습니다.

모든 서비스 화면에는 다음 고지가 노출됩니다.
> "본 서비스는 민간 베타 서비스이며 법무부·외교부·법제처의 공식 채널이 아닙니다. 제공되는 정보는 출입국 매뉴얼(2026.04.14) 및 국가법령정보(`law.go.kr`)를 인용한 일반 정보이며, 법적 효력이 없습니다. 개별 사안은 반드시 1345 또는 관할 출입국·외국인청에 확인하십시오."

### 9.2 승인·거부 단정 금지
- 사증 발급 가능 여부를 단정하지 않습니다.
- 서류를 구비했다고 해서 발급이 보장된다고 기술하지 않습니다.
- 심사관의 재량 판단 가능성을 모든 절차 안내에서 명시합니다.

### 9.3 데이터 노후화 위험 관리
- 매뉴얼 캡처 일자(2026.04.14)를 모든 안내 카드에 노출합니다.
- 법령 개정 시 해당 항목에 "갱신 검토 필요" 플래그를 부착합니다.
- 중기 로드맵: `open.law.go.kr` API를 통한 법령 개정 자동 감지

### 9.4 "공식 기관 사칭" 방지
- "민간 베타 서비스 / 비공식" 배지를 메인 페이지 상단에 노출합니다.
- UI 색상·로고가 법무부·하이코리아·출입국청 공식 색채와 혼동되지 않도록 관리합니다.

## 10. 개인정보·민원대행 회피 원칙

### 10.1 개인정보 비수집
본 서비스는 다음 정보를 수집하지 않습니다.

- 사용자 성명, 국적, 등록번호, 여권번호
- 개별 체류자격 서류 원본
- 신청서·서식 작성 대행

사용자가 자연어 입력에서 개인 정보를 스스로 기재한 경우, AI는 해당 정보를 처리에 사용하나 저장·전달하지 않습니다.

### 10.2 민원대행 미제공
본 서비스는 다음 행위를 수행하지 않습니다.

- 출입국·외국인청 신청서의 대신 작성·제출
- 사증 발급 신청 대리
- 이의신청·행정심판·소송 지원
- 출입국 심사·조회 시스템 접속

외국인 민원대행이 필요한 경우, 공인된 행정사 또는 변호사에게 의뢰하도록 안내합니다.

### 10.3 유료 서비스·영업 행위 비연계
본 서비스는 특정 유료 서비스, 행정사, 법무사, 이민 컨설턴트를 추천하거나 중개하지 않습니다.

## 12. 기대효과

### 12.1 법령데이터 활용 효과

- **법령의 시민 도달성 제고**: `law.go.kr`·`open.law.go.kr`의 공개 법령 데이터가 외국인이 직접 이해 가능한 형태로 변환됩니다. 공공 법령 데이터 개방의 실제 활용 가치를 입증하는 구체 사례가 됩니다.
- **출처 추적 가능성**: 사용자는 안내 내용이 어떤 법령 조문·매뉴얼 페이지에 근거하는지 확인할 수 있습니다. 정보 신뢰성이 시스템 구조로 보증됩니다.
- **법령 개정 추적 로드맵**: `open.law.go.kr` API 연동을 통해, 법령 개정 시 영향을 받는 안내 항목을 자동으로 감지하고 갱신하는 파이프라인을 구축할 수 있습니다.

### 12.2 행정 접근성 효과

- **언어 장벽 완화**: 행정 한국어로 작성된 법령·매뉴얼을 외국인이 이해 가능한 문장으로 재구성합니다. 다국어 확장(EN/CN/VN)으로 접근성을 단계적으로 넓힙니다.
- **정보 비대칭 감소**: 출입국 정보에 접근하기 어려운 지방·농촌 거주 외국인, 한국어가 부족한 외국인이 동등한 정보 접근권을 가질 수 있습니다.
- **잘못된 신청·미신고 예방**: 절차 안내가 정확해질수록 과태료 부과, 체류자격 상실, 불필요한 재신청이 줄어듭니다.

### 12.3 공공외교 부가 효과

- 한국 체류 외국인의 행정 경험이 개선되면, 한국에 대한 신뢰와 우호 인식이 형성됩니다. 이는 한국을 선택하는 인재·연구자·예술가의 장기 체류 의향에 영향을 미칩니다.
- 공식 기관(법무부·외교부·법제처)의 채널을 대체하지 않고 보완하는 민간 서비스로서, 공공-민간 협력 모델의 사례가 됩니다.

## 15. 예상 우려와 대응

### Q1. "외국인 비자 서비스가 법제처 법령데이터 공모전 취지와 맞나요?"

**대응**: 출입국관리법·시행령·시행규칙·고시는 모두 `law.go.kr`에 공개된 법제처 데이터입니다. 출입국관리법 시행령 별표1의2의 39종 체류자격 분류는 이 공모전이 지향하는 "법령 데이터의 기계판독 가능 활용"의 직접 사례입니다. 외국인이라는 수혜자 집단은 법령 정보 접근에 언어 장벽까지 존재하므로, 공모전 취지 — "법령 데이터의 시민 실생활 기여" — 와 정합성이 높습니다.

### Q2. "AI가 잘못된 법률 정보를 줄 위험이 있지 않나요?"

**대응**: 본 서비스는 RAG(검색 증강 생성) 구조로, AI가 매뉴얼·법령 텍스트에서 검색한 구절만을 컨텍스트로 사용합니다. 검색 결과가 없으면 추정을 생성하지 않습니다. 모든 답변에 출처 인용·캡처 일자·1345 확인 안내가 부착됩니다. 승인·거부를 단정하는 표현은 회피하도록 프롬프트를 구성하고 응답을 점검합니다. "잘못된 법률 정보"의 위험은 *법령 데이터를 인용하지 않을 때* 더 큽니다 — Paradiso는 반대 방향으로 설계되었습니다.

### Q3. "하이코리아·1345가 이미 있는데, 중복 아닌가요?"

**대응**: 하이코리아와 1345는 공식 채널이며, Paradiso는 이를 대체하지 않습니다. Paradiso는 사용자가 1345에 전화하기 전 *자기 상황을 파악하고 질문을 정리하는 단계*를 보조합니다. 또한 1345는 한국어·영어·중국어 등 한정 언어를 지원하며, 야간·휴일에는 응대가 제한됩니다. Paradiso는 24시간, 다국어로 기본 정보를 제공하며 최종 확인은 공식 채널로 연결합니다. 공식 채널과 *상충 없이 보완*하는 포지셔닝입니다.

### Q4. "데이터가 오래되거나 법령이 바뀌면 어떻게 하나요?"

**대응**: 모든 안내에 데이터 기준일(2026.04.14)을 명시합니다. 중기 로드맵으로 `open.law.go.kr` API를 통한 법령 개정 자동 감지를 구현할 계획입니다. 법령·매뉴얼이 개정되면 영향을 받는 항목에 "갱신 검토 필요" 플래그를 부착합니다. 데이터 노후화는 모든 정보 서비스가 안고 있는 문제이며, 이를 *숨기지 않고 사용자에게 명시*하는 것이 Paradiso의 설계 원칙입니다.

### Q5. "개인정보 처리가 우려됩니다."
